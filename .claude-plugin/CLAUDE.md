# Claude Plugin Configuration

## Purpose

Plugin manifest and configuration for integrating the Backlog Toolkit with Claude Code.

## Files

### `plugin.json`
Main plugin manifest defining:
- Plugin metadata (name, version, description)
- Author and repository information
- Skill declarations (commands, paths, descriptions)
- Dependencies and compatibility

**Key sections**:
```json
{
  "name": "backlog-toolkit",
  "version": "1.0.0",
  "skills": [
    {
      "name": "init",
      "command": "/backlog-toolkit:init",
      "path": "commands/init.md",
      "description": "Initialize backlog in any project"
    },
    ...
  ]
}
```

## Plugin Lifecycle

1. **Discovery**: Claude Code scans for `plugin.json`
2. **Loading**: Plugin metadata loaded, skills registered
3. **Activation**: Skills become available as slash commands
4. **Invocation**: User runs command, skill file loaded and executed

## Version Management

Follow semantic versioning (semver):
- **Major** (1.x.x): Breaking changes
- **Minor** (x.1.x): New features, backward compatible
- **Patch** (x.x.1): Bug fixes

Update `plugin.json` version when releasing.

## Skill Registration

Each skill requires:
- **name**: Internal identifier (e.g., "init")
- **command**: User-facing command (e.g., "/backlog-toolkit:init")
- **path**: Relative path to skill file (e.g., "commands/init.md")
- **description**: Short help text shown in command list

## Installation Scopes

Plugin can be installed at different scopes:

### User Scope
Available to user across all projects:
```bash
/plugin install backlog-toolkit --scope user
```

Installed to: `~/.claude/plugins/`

### Project Scope
Available only in current project, shared via `.claude/settings.json`:
```bash
/plugin install backlog-toolkit --scope project
```

Installed to: `.claude/plugins/`

## Development Mode

Test plugin without installing:
```bash
claude --plugin-dir /path/to/backlog-agents
```
