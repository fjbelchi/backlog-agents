# Documentation Directory

## Purpose

Comprehensive operational documentation for the Backlog Toolkit plugin.

## Structure

### `tutorials/`
Step-by-step guides for getting started and daily operations:
- `quickstart-cloud-only.md`: End-to-end onboarding
- `daily-flow.md`: Common workflows and patterns

### `reference/`
Technical reference documentation:
- `command-reference.md`: All available commands and options
- `config-schema.md`: backlog.config.json schema documentation
- `litellm-proxy-config.md`: Production proxy configuration
- `model-costs.md`: Auto-generated model pricing table

### `runbooks/`
Operational procedures for common scenarios:
- `cost-incident-response.md`: Handle budget overruns
- `cache-optimization.md`: Improve prompt cache efficiency
- `batch-operations.md`: Run non-interactive batch jobs
- `ticket-recovery.md`: Recover from failed implementations

### `skills/`
Detailed documentation for each skill:
- One .md file per skill (init, ticket, refinement, implementer)
- Includes planned skills (orchestrator, cost-governor, batch-operator, cache-optimizer)

### `architecture/`
Design decisions and architectural documentation:
- `adr/`: Architecture Decision Records
- `agent-coordination.md`: How agents communicate and coordinate
- `quality-gates.md`: Gate design and rationale

### `troubleshooting/`
Common issues and solutions:
- Installation problems
- Configuration errors
- Skill invocation issues

### `contributing/`
Guidelines for contributors:
- Development setup
- Testing procedures
- Documentation standards

## Maintenance

### Auto-Generated Files
These files are generated by scripts and should not be manually edited:
- `reference/model-costs.md` (via `scripts/docs/generate-model-table.sh`)
- `reference/config-schema.md` (via `scripts/docs/generate-config-reference.py`)

### Validation
Run these scripts to ensure documentation quality:
```bash
./scripts/docs/check-links.sh         # Check for broken links
./scripts/docs/verify-snippets.sh     # Validate code snippets
./scripts/docs/check-doc-coverage.py  # Ensure all features documented
```

## Writing Guidelines

1. **Be concise**: Prefer examples over lengthy explanations
2. **Show, don't tell**: Include command examples and expected output
3. **Link liberally**: Cross-reference related docs
4. **Keep it current**: Update docs when changing features
