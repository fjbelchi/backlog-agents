## Quick Start for Contributors

1. **Fork and clone**:
```bash
git clone https://github.com/[your-username]/backlog-agents
cd backlog-agents
```

2. **Install for development**:
```bash
./install.sh --local --force
```

3. **Run tests**:
```bash
./tests/test-config-schema.sh
./tests/test-templates.sh
./tests/test-install.sh
```

4. **Make changes**:
- Create feature branch: `git checkout -b feature/your-feature`
- Make changes
- Test thoroughly
- Update docs

5. **Validate**:
```bash
./scripts/docs/check-links.sh
./scripts/docs/verify-snippets.sh
./scripts/docs/check-doc-coverage.py
```

6. **Submit PR**:
- Push to your fork
- Create pull request with clear description
- Wait for review

## Contribution Types

### Bug Fixes
- Include test that reproduces the bug
- Explain root cause in PR description
- Update CHANGELOG.md

### New Features
- Discuss in GitHub issue first (for large features)
- Add tests
- Add documentation
- Update relevant examples/tutorials

### Documentation
- Fix typos/errors
- Improve clarity
- Add examples
- Translate (if applicable)

### Skill Development
- Follow skill template
- Include SKILL.md in skill directory
- Add tests
- Document in `docs/skills/`
- Update plugin manifest

## Review Process

1. **Automated checks**: CI runs tests, linting, doc validation
2. **Maintainer review**: Code quality, design, test coverage
3. **Documentation review**: Docs updated, examples work
4. **Approval**: At least one maintainer approval required
5. **Merge**: Squash merge to main with conventional commit message

## Code of Conduct

Be respectful, constructive, and collaborative. See full code of conduct in `CONTRIBUTING.md`.
