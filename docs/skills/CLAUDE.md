# Skills Documentation Directory

## Purpose

Detailed documentation for each skill (active and planned) in the Backlog Toolkit.

## Documentation Files

### Active Skills

#### `backlog-init.md`
Bootstraps backlog structure in any project.

**Key features**:
- Stack detection (TypeScript, Python, Go, Rust, etc.)
- Directory structure creation
- Template generation
- Config file initialization with quality gates

**Related**: `skills/backlog-init/SKILL.md`

#### `backlog-ticket.md`
Generates validated, cost-estimated tickets from descriptions.

**Key features**:
- Codebase context analysis
- Template filling with real content
- 6 validation checks
- Cost estimation (Opus, Sonnet, Haiku)

**Related**: `skills/backlog-ticket/SKILL.md`

#### `backlog-refinement.md`
Maintains backlog health and consistency.

**Key features**:
- Code reference validation
- Duplicate detection
- Completeness checks
- Health scoring (0-100)
- Obsolete ticket cleanup

**Related**: `skills/backlog-refinement/SKILL.md`

#### `backlog-implementer.md`
Orchestrates implementation with Agent Teams and quality gates.

**Key features** (v7.0):
- Smart agent routing (file pattern matching)
- Embedded skill catalog (7 disciplines)
- Configurable review pipeline (2-4 reviewers)
- 5-gate quality pipeline
- Cost tracking
- Two Iron Laws: Commit-Before-Move, Zero-Hacks

**Related**: `skills/backlog-implementer/SKILL.md`

## Documentation Structure

Each skill doc includes:

```markdown
# Skill: Name

## Overview
High-level description and purpose

## When to Use
Specific scenarios and use cases

## Command Syntax
How to invoke the skill

## Configuration
Relevant config sections

## Examples
Common usage patterns with expected output

## Troubleshooting
Common issues and solutions

## Implementation Details
How the skill works internally

## Related
Links to other docs
```

## Updating Skill Docs

When changing a skill:
1. Update SKILL.md in `skills/` (source of truth)
2. Update user-facing docs in `docs/skills/`
3. Update command docs in `commands/`
4. Update README.md if needed
5. Add changelog entry

## Cross-References

Each skill doc should link to:
- Configuration reference (relevant sections)
- Related skills (dependencies, workflows)
- Troubleshooting guide
- Relevant runbooks

## Related Documentation

- `skills/`: Actual skill implementations (SKILL.md files)
- `commands/`: Slash command definitions
- `docs/tutorials/`: Usage guides
- `docs/reference/command-reference.md`: Command syntax
