## Writing Tutorials

### Best Practices

1. **Test everything**: Run all commands yourself before publishing
2. **Show actual output**: Don't use placeholders, show real results
3. **Explain why**: Don't just list commands, explain the reasoning
4. **Handle errors**: Include troubleshooting for common failures
5. **Be specific**: Avoid vague instructions like "configure as needed"

### Common Pitfalls

- Assuming too much prior knowledge
- Skipping "obvious" steps
- Not testing on clean environment
- Outdated screenshots or output
- Missing error handling
