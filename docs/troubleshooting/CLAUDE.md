# Troubleshooting Directory

## Purpose

Solutions to common issues users encounter when using the Backlog Toolkit.

## Structure

Each troubleshooting document covers a specific problem category:

### Installation Issues
- Plugin not found after install
- Permission errors
- Dependency conflicts
- Version mismatches

### Configuration Errors
- Invalid `backlog.config.json`
- Schema validation failures
- Missing required fields
- Type mismatches

### Skill Invocation Problems
- Command not recognized
- Permission denied
- Skill fails to load
- Wrong version executed

### Ticket Validation Failures
- Template section missing
- Invalid frontmatter YAML
- Acceptance criteria count < minimum
- File references don't exist

### Implementation Issues
- Quality gate failures (lint, test, review)
- Agent coordination errors
- Cost budget exceeded
- Partial implementation left in inconsistent state

### LiteLLM Integration
- Proxy connection refused
- Authentication failures
- Rate limiting
- Model not available

## Quick Reference

### Most Common Issues

1. **"Command not found: /backlog-toolkit:init"**
   → Plugin not installed. Run: `/plugin install --path /path/to/backlog-agents`

2. **"Invalid config: missing required field 'project.stack'"**
   → Run backlog-init to generate valid config: `/backlog-toolkit:init`

3. **"Quality gate failed: lint errors"**
   → Fix linting issues: `npm run lint -- --fix` or equivalent for your stack

4. **"Cost budget exceeded"**
   → Check spending: `./scripts/ops/cost_guard.py --ledger .backlog-ops/usage-ledger.jsonl`

5. **"Agent failed to connect"**
   → Verify LiteLLM proxy running: `curl http://localhost:8000/health`

## Adding New Troubleshooting Docs

1. Identify recurring issue (check GitHub issues, support requests)
2. Document symptoms clearly
3. Test all solutions before publishing
4. Include prevention tips
5. Cross-reference related docs

## Escalation Path

If troubleshooting docs don't help:
1. Check GitHub issues: https://github.com/[org]/backlog-agents/issues
2. Review runbooks: `docs/runbooks/`
3. Ask in community forum/Slack
4. File new GitHub issue with:
   - Symptom description
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment (OS, Claude Code version, plugin version)
   - Relevant config/logs

- `docs/reference/`: Technical references
- `docs/tutorials/`: Getting started guides
