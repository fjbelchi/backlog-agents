# Scripts Directory

## Purpose

Utility scripts supporting the backlog toolkit. Organized by function area.

## Directories

### `ops/`
Operational tools for cloud-first deployments:
- `cost_guard.py`: Monitor token usage and enforce budgets
- `sync-model-registry.sh`: Refresh model pricing and availability
- `batch_submit.py`: Submit tickets for async batch processing
- `batch_reconcile.py`: Reconcile batch results with backlog
- `prompt_prefix_lint.py`: Validate prompt prefixes for cache efficiency

### `docs/`
Documentation generation and validation:
- `generate-model-table.sh`: Create pricing table from model registry
- `generate-config-reference.py`: Build config schema documentation
- `check-links.sh`: Validate internal and external links
- `verify-snippets.sh`: Test code snippets in documentation
- `check-doc-coverage.py`: Ensure all features are documented

## Usage Patterns

### During Development
```bash
# Validate documentation changes
./scripts/docs/check-links.sh
./scripts/docs/verify-snippets.sh

# Test schema changes
./scripts/docs/generate-config-reference.py
```

### In Production
```bash
# Monitor costs
./scripts/ops/cost_guard.py --ledger .backlog-ops/usage-ledger.jsonl

# Update model data
./scripts/ops/sync-model-registry.sh
./scripts/docs/generate-model-table.sh

# Run batch operations
./scripts/ops/batch_submit.py --input backlog/data/pending/*.md
./scripts/ops/batch_reconcile.py --status completed
```
