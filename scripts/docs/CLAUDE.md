# Documentation Scripts

## Purpose

Scripts for generating, validating, and maintaining documentation.

## Scripts

### `generate-config-reference.py`
Auto-generate configuration schema documentation.

**Usage**:
```bash
./scripts/docs/generate-config-reference.py
```

**Features**:
- Parse config schema from `config/schema.json`
- Generate markdown reference doc
- Include field types, defaults, examples
- Write to `docs/reference/config-schema.md`

**Dependencies**: Python 3.10+, `json`, `argparse`

**When to run**: After changing config schema

### `generate-model-table.sh`
Create model pricing table from registry.

**Usage**:
```bash
./scripts/docs/generate-model-table.sh
```

**Features**:
- Read `.backlog-ops/model-registry.json`
- Format as markdown table
- Include pricing, context windows, rate limits
- Write to `docs/reference/model-costs.md`

**Dependencies**: `jq`, `bash`

**When to run**: After `sync-model-registry.sh`

### `check-links.sh`
Validate internal and external documentation links.

**Usage**:
```bash
# Check all docs
./scripts/docs/check-links.sh

# Check specific directory
./scripts/docs/check-links.sh docs/tutorials/
```

**Features**:
- Find all markdown links
- Check internal file references
- Test external URLs (optional)
- Report broken links
- Exit non-zero if failures found

**Dependencies**: `bash`, `grep`, `curl` (for external links)

**When to run**: Before committing doc changes, in CI

### `verify-snippets.sh`
Test code snippets in documentation.

**Usage**:
```bash
# Test all snippets
./scripts/docs/verify-snippets.sh

# Test specific file
./scripts/docs/verify-snippets.sh docs/tutorials/quickstart-cloud-only.md
```

**Features**:
- Extract code blocks from markdown
- Execute bash/python snippets
- Verify expected output
- Report failures with line numbers

**Dependencies**: `bash`, `python3`, `sed`, `awk`

**When to run**: Before committing tutorials/examples

### `check-doc-coverage.py`
Ensure all features are documented.

**Usage**:
```bash
./scripts/docs/check-doc-coverage.py
```

**Features**:
- Parse skill files for public features
- Check if documented in `docs/`
- Find undocumented commands/options
- Generate coverage report

## CI Integration

Add to `.github/workflows/docs.yml`:

```yaml
name: Validate Docs

on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Check links
        run: ./scripts/docs/check-links.sh

      - name: Verify snippets
        run: ./scripts/docs/verify-snippets.sh

      - name: Check coverage
        run: ./scripts/docs/check-doc-coverage.py
```

## Workflow

### After Changing Config Schema
```bash
./scripts/docs/generate-config-reference.py
git add docs/reference/config-schema.md
git commit -m "docs: update config reference"
```

### After Model Registry Update
```bash
./scripts/ops/sync-model-registry.sh
./scripts/docs/generate-model-table.sh
git add docs/reference/model-costs.md
git commit -m "docs: update model pricing"
```

### Before Committing Docs
```bash
./scripts/docs/check-links.sh
./scripts/docs/verify-snippets.sh
./scripts/docs/check-doc-coverage.py
```

## Error Handling

All scripts:
- Exit with status 0 on success, non-zero on failure
- Write errors to stderr
- Include line numbers for failures
- Support `--help` flag
