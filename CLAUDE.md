# Backlog Toolkit - Project Overview

## Purpose

This is a reusable Claude Code plugin for backlog management. It provides 4 main skills for creating, refining, and implementing project tickets with built-in cost estimation and quality gates.

## Key Components

### Skills (`skills/`)
- **backlog-init**: Bootstraps backlog structure in any project
- **backlog-ticket**: Generates validated tickets with cost estimates
- **backlog-refinement**: Maintains backlog health and consistency
- **backlog-implementer**: Orchestrates implementation with Agent Teams and quality gates

### Commands (`commands/`)
Slash command definitions that invoke the skills above via `/backlog-toolkit:*`

### Configuration (`config/`)
- Schema and presets for `backlog.config.json`
- LiteLLM proxy configuration templates
- Agent routing and review pipeline configuration

### Templates (`templates/`)
Markdown templates for 4 ticket types: TASK, BUG, FEAT, IDEA

### Scripts (`scripts/`)
- **ops/**: Cost tracking, model registry, batch operations
- **docs/**: Documentation generation and validation
- **ticket/**, **refinement/**, **implementer/**: Skill-specific utilities

### Documentation (`docs/`)
- `tutorials/`: Quickstart guides and daily workflows
- `reference/`: Command reference, config schema, API docs
- `runbooks/`: Operational procedures (cost incidents, cache optimization)
- `skills/`: Detailed skill documentation
- `architecture/`: ADRs and design decisions

### Tests (`tests/`)
Validation scripts for config schema, templates, and installation

## Development Workflow

1. **Install plugin**: `/plugin install --path .`
2. **Test changes**: Run scripts in `tests/`
3. **Validate docs**: Run scripts in `scripts/docs/`
4. **Update version**: Bump version in plugin.json and affected SKILLs

## Key Principles

- **Stack-agnostic**: Detects project type and adapts
- **Cost-aware**: All operations include cost estimation and tracking
- **Quality-gated**: 5 gates (Plan, TDD, Lint, Review, Commit)
- **Config-driven**: Behavior controlled via `backlog.config.json`

## Important Files

- `README.md`: User-facing documentation
- `.claude-plugin/plugin.json`: Plugin manifest
- `install.sh`: Manual installation script
- `backlog.config.json`: Generated by init, controls toolkit behavior
