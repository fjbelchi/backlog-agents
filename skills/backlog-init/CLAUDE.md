# Backlog Init Skill

## Purpose

Bootstraps backlog infrastructure in any project. Detects project stack, creates directory structure, copies templates, and generates `backlog.config.json` with appropriate quality gates.

## Key Features

1. **Stack Detection**: Automatically identifies project type (TypeScript, Python, Go, Rust, generic)
2. **Directory Creation**: Creates `backlog/data/{pending,in-progress,completed}` and `backlog/templates`
3. **Template Distribution**: Copies 4 ticket templates (TASK, BUG, FEAT, IDEA)
4. **Config Generation**: Creates `backlog.config.json` from stack-appropriate preset

## Stack Detection Logic

```
1. Check for package.json + tsconfig.json â†’ typescript
2. Check for pyproject.toml or requirements.txt â†’ python
3. Check for go.mod â†’ go
4. Check for Cargo.toml â†’ rust
5. Check for multiple or no indicators â†’ generic
```

## Configuration Presets

Uses presets from `config/presets/` based on detected stack:
- `typescript.json`: For Node.js/TypeScript projects
- `python.json`: For Python projects
- `go.json`: For Go projects
- `rust.json`: For Rust projects
- `generic.json`: Fallback for unknown stacks

Each preset includes stack-appropriate:
- Quality gate commands (lint, test, type check, build)
- Agent routing rules (file patterns â†’ agent types)
- Review pipeline configuration

## Invocation

```bash
/backlog-toolkit:init
```

No arguments required. The skill:
1. Detects current directory and stack
2. Prompts for confirmation before creating files
3. Creates structure and generates config
4. Reports success and next steps

## Error Handling

- **Already initialized**: Warns if `backlog/` or `backlog.config.json` already exists
- **Permission denied**: Reports if unable to create directories
- **Unknown stack**: Falls back to generic preset
- **Missing templates**: Reports and suggests reinstalling plugin

## Post-Initialization

After running init, users should:
1. Review and customize `backlog.config.json`
2. Test quality gates: `npm test`, `npm run lint`, etc.
3. Create first ticket: `/backlog-toolkit:ticket "description"`
