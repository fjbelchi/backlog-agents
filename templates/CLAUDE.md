# Templates Directory

## Purpose

Markdown templates for the four ticket types supported by the Backlog Toolkit.

## Template Files

### `task-template.md`
General implementation work template.

**Sections**:
- Context: Why this task exists
- Description: What needs to be done
- Affected Files: Files that will be modified
- Acceptance Criteria: Definition of done (min 3)
- Test Strategy: How to verify it works
- Dependencies: Required tickets or external dependencies

**Frontmatter**:
```yaml
---
id: TASK-NNN
title: Brief title
status: pending
priority: medium
created: YYYY-MM-DD
updated: YYYY-MM-DD
assignee: unassigned
blockers: []
depends_on: []
shared_files: []
related_docs: []
---
```

### `bug-template.md`
Defect report and fix template.

**Additional sections** (beyond TASK):
- Severity: critical, high, medium, low
- Environment: Where the bug occurs (prod, staging, dev)
- Steps to Reproduce: How to trigger the bug
- Expected vs Actual Behavior

**Use for**: Bugs, defects, regressions

### `feature-template.md`
New feature development template.

**Additional sections**:
- Phase: plan, implement, test, deploy
- User Stories: User-facing value proposition
- Success Metrics: How to measure feature success

**Use for**: New features, major enhancements

### `idea-template.md`
Exploratory idea and experiment template.

**Additional sections**:
- Type: experiment, spike, research, prototype
- ICE Score: Impact, Confidence, Ease (scored 1-10 each)
- Hypothesis: What we're trying to validate
- Success Criteria: How we'll know if it worked
- Learnings: Post-experiment insights

**Use for**: Experiments, spikes, research, prototypes

## Template Usage

Templates are used by:
1. **backlog-init**: Copies templates to `backlog/templates/` in user project
2. **backlog-ticket**: Fills templates with actual content when creating tickets

## Template Structure Rules

All templates must have:
1. **YAML frontmatter**: Ticket metadata
2. **Required sections**: As defined in `backlog.config.json`
3. **Placeholder text**: Guiding comments for manual ticket creation
4. **Markdown formatting**: Valid CommonMark syntax

## Customizing Templates

Users can customize templates in their project's `backlog/templates/` directory after running `backlog-init`. Changes are project-specific.

To modify plugin defaults:
1. Edit template in this directory
2. Test with `./tests/test-templates.sh`
3. Update `backlog.config.json` schema if adding sections
4. Document changes in this CLAUDE.md

## Validation

Templates are validated by:
- **Schema check**: Frontmatter matches expected fields
- **Section check**: All required sections present
- **Format check**: Valid YAML and Markdown
- **Test script**: `tests/test-templates.sh`

Run validation:
```bash
./tests/test-templates.sh
```

## Template Versioning

When making breaking changes to templates:
1. Bump plugin version
2. Document migration path
3. Provide upgrade script if needed
4. Update all 4 templates consistently
