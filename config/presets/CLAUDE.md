# Configuration Presets

## Purpose

Pre-configured `backlog.config.json` templates for common project stacks. Used by `backlog-init` to generate stack-appropriate configurations.

## Available Presets

### `typescript.json`
For Node.js/TypeScript projects (React, Next.js, Vue, Angular, etc.)

**Quality gates**:
- Type check: `npx tsc --noEmit`
- Lint: `npx eslint .`
- Test: `npx vitest run` or `npm test`
- Build: `npx tsc` or `npm run build`

**Agent routing**:
- `**/*.tsx`, `**/*.jsx` â†’ frontend
- `**/api/**/*.ts` â†’ backend
- `**/*.test.ts` â†’ test-engineer

### `python.json`
For Python projects (Django, Flask, FastAPI, etc.)

**Quality gates**:
- Type check: `mypy .`
- Lint: `ruff check .`
- Test: `pytest`
- Build: N/A (interpreted language)

**Agent routing**:
- `**/api/**/*.py` â†’ backend
- `**/models/**/*.py` â†’ backend
- `**/tests/**/*.py` â†’ test-engineer
- `**/*.ipynb` â†’ ml-engineer

### `go.json`
For Go projects

**Quality gates**:
- Type check: `go build ./...` (Go has built-in type checking)
- Lint: `golangci-lint run`
- Test: `go test ./...`
- Build: `go build`

**Agent routing**:
- `**/*.go` â†’ backend
- `**/Dockerfile` â†’ devops
- `**/*_test.go` â†’ test-engineer

### `rust.json`
For Rust projects

**Quality gates**:
- Type check: `cargo check`
- Lint: `cargo clippy -- -D warnings`
- Test: `cargo test`
- Build: `cargo build --release`

**Agent routing**:
- `**/*.rs` â†’ backend
- `**/tests/**/*.rs` â†’ test-engineer
- `**/benches/**/*.rs` â†’ performance

### `generic.json`
Fallback for unknown or multi-language projects

**Quality gates**:
- Minimal gates, user must customize
- Basic linting and testing placeholders

## Preset Selection Logic

In `backlog-init`:

```
1. Check for package.json + tsconfig.json â†’ typescript
2. Check for pyproject.toml or requirements.txt â†’ python
3. Check for go.mod â†’ go
4. Check for Cargo.toml â†’ rust
5. Default â†’ generic
```

## Customizing Presets

When maintaining presets:
1. Ensure commands work for typical project setup
2. Use commonly installed tools (prefer built-in over exotic)
3. Include sensible agent routing defaults
4. Test on fresh project of that stack
5. Document any assumptions (e.g., "assumes npm installed")

## Adding New Presets

1. Create `stack-name.json` with full config structure
2. Test on real project of that stack
3. Update selection logic in `backlog-init`
4. Add to this CLAUDE.md
5. Document in `docs/reference/config-schema.md`
